# TUI Manager для P2P приложения

Пакет `tuimanager` предоставляет полнофункциональный текстовый интерфейс (TUI) для управления пиром в распределённой системе.

## Обзор

TUI менеджер позволяет:
- 📡 **Обнаруживать пиры** в сети через multicast/UDP
- 🔌 **Подключаться к пирам** вручную по адресу IP:port
- 📊 **Просматривать активные сессии** с информацией о типе и статусе
- 📋 **Управлять подключениями** - создание и закрытие сессий
- ℹ️ **Получать подробную информацию** о выбранных пирах и сессиях

## Архитектура

TUI менеджер построен на основе архитектурного паттерна **TEA (The Elm Architecture)** с использованием библиотеки [bubbletea](https://github.com/charmbracelet/bubbletea).

### The Elm Architecture

```
┌─────────────┐
│   Model     │
│  (состояние) │
└──────┬──────┘
       │
       ▼
┌─────────────┐     ┌─────────────┐
│    View     │◄────┤   Update    │
│ (отрисовка) │     │  (логика)   │
└─────────────┘     └──────┬──────┘
                           │
                           ▼
                    ┌─────────────┐
                    │    Cmds     │
                    │  (эффекты)  │
                    └─────────────┘
```

### Основные компоненты

1. **Model** - содержит всё состояние приложения:
   - Списки пиров и сессий
   - Выбранные элементы
   - Статус и ошибки
   - Состояние модальных окон

2. **Update** - обрабатывает сообщения (Msg):
   - Нажатия клавиш
   - Загрузку данных
   - Таймеры обновления

3. **View** - рендерит UI:
   - Двухколоночный макет
   - Панели с информацией
   - Модальные окна

4. **Commands** - выполняют побочные эффекты:
   - Загрузка из хранилищ
   - Обнаружение пиров
   - Таймеры

## Структура интерфейса

```
╭─────────────────────────────────╮╭─────────────────────────────────╮
│     Обнаруженные пиры           ││     Активные сессии             │
│   ─────────────────────────     ││   ─────────────────────────     │
│   > Peer1                       ││   > Session1                    │
│     192.168.1.1:8001            ││     Входящее | Открыто          │
│     Peer2                       ││     Session2                    │
│     192.168.1.2:8001            ││     Исходящее | Открыто         │
│                                 ││                                 │
╰─────────────────────────────────╯╰─────────────────────────────────╯

╭────────────────────────────────────────────────────────────────────╮
│  Имя: Peer1                                                        │
│  ID: abc123...                                                     │
│  IP: 192.168.1.1                                                   │
╰────────────────────────────────────────────────────────────────────╯

Статус: Пиров обнаружено: 2

Q: Выход | R: Обновить | C: Подключиться | D: Отключить | I: Инфо | H: Помощь
```

## Горячие клавиши

| Клавиша | Действие |
|---------|----------|
| `Q` / `Ctrl+C` | Выход из приложения |
| `R` | Принудительно обновить списки |
| `C` | Подключиться к пиру (открывает форму ввода) |
| `D` | Отключить выбранную сессию |
| `I` | Показать информацию о выбранном элементе |
| `H` | Показать справку по горячим клавишам |
| `Tab` | Переключение между панелями |
| `↑/↓` или `J/K` | Навигация по списку |
| `Enter` | Выбрать элемент |
| `Escape` | Закрыть форму/модальное окно |

## Использование

### Создание менеджера

```go
import (
    "github.com/first-debug/p2p/internal/manager/TuiManager"
    peerstorage "github.com/first-debug/p2p/internal/storage/peer-storage"
    sessionstorage "github.com/first-debug/p2p/internal/storage/session-storage"
)

// Создаем хранилища
peerStore := inmemorypeerstorage.NewMemoryPeerStorage()
sessionStore := inmemorysessionstorage.NewMemorySessionStorage()

// Создаем explorer для обнаружения пиров
explorer := updexplorer.NewUDPExplorer(cfg, peerInfo, peerStore)

// Создаем TUI менеджер
manager, err := tuimanager.NewTuiManager(explorer, peerStore, sessionStore)
if err != nil {
    log.Fatal(err)
}

// Запускаем менеджер (блокирующий вызов)
if err := manager.Run(); err != nil {
    log.Fatal(err)
}
```

### Остановка менеджера

```go
// Создаем контекст для остановки
ctx, cancel := context.WithTimeout(context.Background(), 5*time.Second)
defer cancel()

// Останавливаем менеджер
manager.Stop(ctx)
```

```
┌─────────────────┐
│   TuiManager    │
│   (bubbletea)   │
└────────┬────────┘
         │
    ┌────┴────┬────────────┬──────────────┐
    │         │            │              │
    ▼         ▼            ▼              ▼

│Explorer│ │PeerStorage│ │SessionStorage│ │Server   │
```

- **Explorer** - используется для обнаружения пиров в сети

## Расширение функциональности

### Добавление новых сообщений

```go
// Создаем новый тип сообщения
type customMsg struct {
    data string
}

// Отправляем сообщение в update
func doSomething() tea.Cmd {
    return func() tea.Msg {
        return customMsg{data: "hello"}
    }
}
```

### Добавление обработки сообщений

```go
func (m model) Update(msg tea.Msg) (tea.Model, tea.Cmd) {
    switch msg := msg.(type) {
    case customMsg:
        // Обрабатываем сообщение
        m.status = msg.data
        return m, nil
    }
    // ...
}
```

### Добавление новых горячих клавиш

```go
// Определяем клавишу в keyMap
var keyMap = struct {
    Custom key.Binding
}{
    Custom: key.NewBinding(
        key.WithKeys("x"),
        key.WithHelp("X", "Действие"),
    ),
}

// Обрабатываем в update
func (m model) Update(msg tea.Msg) (tea.Model, tea.Cmd) {
    if keyMsg, ok := msg.(tea.KeyMsg); ok {
        if key.Matches(keyMsg, keyMap.Custom) {
            m.doSomething()
            return m, nil
        }
    }
    // ...
}
```

## Отладка

Логгер TUI менеджера выводит сообщения в stderr с префиксом `[TUIManager]`:

```
[TUIManager] 2024/01/01 12:00:00 Попытка подключения к адресу: 192.168.1.100:8001
```

Для включения подробного логирования добавьте вывод stderr в файл:

```bash
./application 2> tui.log
```

## Отличия от версии на tview

| Характеристика | tview (старая) | bubbletea (новая) |
|---------------|----------------|-------------------|
| Архитектура | Императивная | Функциональная (TEA) |
| Управление состоянием | Мутабельное | Иммутабельное |
| Тестируемость | Сложная | Упрощенная |
| Компонуемость | Низкая | Высокая |
| Зависимости | tview, tcell | bubbletea, bubbles, lipgloss |

## Известные ограничения

1. **Приведение типов сессий**: Для получения информации о пире из сессии используется type assertion. В будущем рекомендуется добавить метод `GetPeer()` в интерфейс `session.Session`.

2. **Подключение к пирам**: Логика фактического подключения к пирам должна быть реализована отдельно. В текущей версии отображается только статус подключения.

3. **Тестирование**: TUI компоненты сложно тестировать автоматически. Рекомендуется ручное тестирование основных сценариев.
